# 06-05 — Modo sem Aposta como Padrão

**Owner:** CTO (delegado)
**Anel:** 1 (urgente — pode começar agora)
**Depende de:** N/A
**Reqs:** NFR-LEGAL-01 (parcial)

## Objetivo

Garantir que o modo padrão do PvP seja sem aposta financeira (glória/ranking apenas), com aposta como recurso opt-in.

## Entregas

- [ ] **Modo padrão:** PvP por ranking/ELO sem stake
- [ ] **Aposta como opt-in:** toggle/modal explícito para partidas com stake
- [ ] **Confirmação adicional:** segunda tela de confirmação antes de partida com aposta
- [ ] **Bloqueio por jurisdição:** partidas com stake bloqueadas onde for ilegal (integração com 06-02)
- [ ] **Limite de aposta:** valor mínimo e máximo configurável
- [ ] **Dry-run mode:** partidas com stake simuladas (sem dinheiro real) para teste

## Critérios de aceite

- Novo usuário entra no PvP e vê apenas modo ranking
- Aposta requer ação consciente (toggle + confirmação)
- Partida com stake bloqueada se jurisdição não permitir
- Limites de aposta respeitados on-chain e off-chain

## Abordagem técnica

### Unity (cliente)
1. Tela de PvP: modo "Ranked" (padrão, destacado) e modo "Staked" (secundário, com cadeado se bloqueado)
2. Modal de confirmação ao selecionar "Staked" com valor e warning
3. Tooltip explicando riscos

### Backend
1. Endpoint `POST /matchmaking` com parâmetro `stakeAmount` (0 = ranked)
2. Validação de geofencing + idade antes de permitir stake
3. Limites de aposta por faixa de ELO

### Contratos
1. MatchEscrow já suporta `stake == 0` (partida sem aposta) — verificar
2. Se não, adicionar validação `stakeAmount >= minStake` apenas quando `isStaked == true`

## Risco

- Baixo — é principalmente mudança de UX + validação backend
- Contratos já devem suportar stake=0; verificar antes de modificar
