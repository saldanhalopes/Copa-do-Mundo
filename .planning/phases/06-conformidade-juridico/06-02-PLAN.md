# 06-02 — Geofencing (Bloqueio Regional)

**Owner:** CTO (delegado)
**Anel:** 2
**Depende de:** N/A
**Reqs:** NFR-LEGAL-01

## Objetivo

Bloquear compra de pacotes e apostas PvP em jurisdições onde loot boxes/staking são proibidos ou restritos.

## Entregas

- [ ] **Geofencing de pacotes:** bloquear compra de pacotes pagos em Bélgica, Holanda e outras jurisdições restritivas
- [ ] **Geofencing de PvP com stake:** bloquear criação/aceitação de partidas com aposta em jurisdições onde gambling é regulado
- [ ] **Mecanismo de detecção:** IP geolocation (Cloudflare + fallback MaxMind), + verificação de país no cadastro
- [ ] **Tratamento de VPN:** flag de risco, exigir documento se detectado
- [ ] **Fallback on-chain:** wallet-level country flag forçado via contrato (override emergencial)

## Critérios de aceite

- Usuário de país bloqueado não consegue comprar pacote nem apostar
- Usuário de país liberado consegue operar normalmente
- Bloqueio funcional tanto no cliente Unity quanto na Web
- Mapa de jurisdições configurável (JSON externo, sem redeploy)
- Logs de tentativas bloqueadas para auditoria

## Abordagem técnica

1. **Cloudflare IP geolocation** no backend (CF-IPCountry header)
2. **Lista de jurisdições bloqueadas** em arquivo configurável `blocked-jurisdictions.json`
3. **Middleware de geofencing** no backend para endpoints de compra/criação de partida
4. **Verificação dupla** no contrato inteligente (emergency country block via admin)
5. **Clients:** bloquear UX antes de chamar o backend (evitar gás desperdiçado)

## Risco

- VPNs podem contornar IP geolocation → mitigar com KYC + flag de risco
- Não deve bloquear jurisdições acidentalmente (testar com lista de IPs conhecidos)
