# 06-03 — KYC/AML (On-Ramp e Marketplace)

**Owner:** CTO (delegado)
**Anel:** 2
**Depende de:** N/A
**Reqs:** NFR-LEGAL-02

## Objetivo

Garantir conformidade com KYC/AML para compras fiat e operações no marketplace secundário.

## Entregas

- [x] **KYC delegado ao provedor de pagamento:** Crossmint, MoonPay ou Binance Pay realizam KYC do usuário
- [x] **Limites de transação sem KYC:** configurar valores máximos antes de exigir verificação (varia por jurisdição)
- [x] **Monitoramento de transações suspeitas:** alertas para padrões anômalos
- [x] **Anti-wash trading:** detecção de vendas circulares no marketplace
- [x] **Registro de AML:** manter logs de transações para auditoria regulatória

## Critérios de aceite

- Usuário sem KYC não consegue comprar acima do limite estabelecido
- Provedor de pagamento confirmado com integração funcional
- Logs de transações disponíveis para consulta
- Wash trading detection report gerado periodicamente

## Abordagem técnica

1. **Crossmint/MoonPay** — já têm KYC embutido. Integrar via SDK.
2. **Binance Pay** — KYC já feito pela Binance. O usuário precisa ter conta Binance verificada.
3. **Camada própria:** apenas para transações P2P acima de threshold.
4. **Backend:** manter tabela `kyc_status` por wallet address.
5. **Marketplace:** adicionar rate limiting + detecção de padrão de wash trading (mesmo IP comprando e vendendo para si mesmo)

## Risco

- KYC é uma barreira de conversão. Balancear com limites baixos sem KYC para usuários casuais.
- Delegar ao provedor reduz responsabilidade legal, mas ainda exige due diligence.
