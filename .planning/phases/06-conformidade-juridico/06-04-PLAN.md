# 06-04 — Verificação de Idade

**Owner:** CTO (delegado)
**Anel:** 2
**Depende de:** N/A
**Reqs:** NFR-LEGAL-04

## Objetivo

Impedir que menores de idade comprem pacotes, apostem em PvP ou tenham dados coletados sem consentimento parental.

## Entregas

- [x] **Verificação de idade no cadastro:** data de nascimento obrigatória (`/auth/register`)
- [x] **Bloqueio de PvP com stake para menores:** idade mínima (18+ configurável) via API + contrato
- [x] **Controles parentais:** limites de compra por faixa etária (via `getPurchaseLimits`) + consentimento parental token-based
- [x] **COPPA compliance:** bloqueio de <13 + fluxo de consentimento parental verificável (`/kyc/parental-consent/*`)
- [ ] **Cross-check:** verificação adicional se idade auto-declarada for suspeita (documento) (requer integração KYC provider)

## Critérios de aceite

- [x] Usuário <18 não consegue participar de partidas com aposta (backend + contrato)
- [x] Usuário <13 não consegue registrar idade (rejeitado no servidor)
- [x] Limites de compra por faixa etária configuráveis (via env var `MIN_AGE_STAKED_PVP`)
- [x] Logs de verificação mantidos para auditoria (coluna `age_verified_at` + `age_verification_method`)

## Abordagem técnica (implementada)

1. **Banco:** colunas `date_of_birth`, `age_verified_at`, `age_verification_method` na tabela `players`
2. **Backend service:** `backend/src/kyc/ageVerification.js` — valida DOB, calcula idade, checa gates
3. **API REST:**
   - `POST /auth/register` — cadastro obrigatório com DOB
   - `POST /auth/register-coppa` — cadastro de <13 com consentimento parental
   - `POST /kyc/age` — registra data de nascimento (valida servidor-side, rejeita <13)
   - `GET /kyc/age-status` — retorna status da verificação e permissões
   - `POST /kyc/parental-consent/init` — inicia fluxo COPPA (envia token)
   - `POST /kyc/parental-consent/confirm` — confirma consentimento parental
   - `GET /kyc/purchase-limits` — limites de compra por faixa etária
4. **Middleware:** `backend/src/middleware/ageGate.js`
   - `requireAge(minAge)` para rotas REST
   - `requireAgeWebSocket(address, minAge)` para fila de matchmaking
5. **Matchmaking:** WebSocket queue bloqueia entrada com `stake > 0` se não verificada
6. **Smart contracts (defense-in-depth):**
   - `MatchEscrow.sol`: `minAge` (state var configurável), `minAgeStakedPvP`, `jurisdictionMinAge`, `AGE_VERIFIER_ROLE`, mapping `ageVerified`, modifier `apenasMaiorIdade`
7. **Chain oracle:** `backend/src/kyc/chainVerification.js` — chama `setAgeVerified` no contrato via ethers.js
8. **Audit logging:** tabela `age_verification_log` com action, age, method, metadata
9. **Parental consent:** tabela `parental_consent` com token-based confirmation
10. **Config:** `MIN_AGE_STAKED_PVP` (env), `AGE_VERIFIER_PRIVATE_KEY`, `CONTRACT_MATCH_ESCROW`
11. **Verificação:** 17 testes passando, contratos compilam sem erro

## Risco

- Auto-declaração é facilmente burlada → considerar KYC de idade como camada adicional para apostas (Ring 3)
- COPPA é complexo e caro de implementar → consultar parecer jurídico antes
- On-chain age verification depende do backend chamar `setAgeVerified` — se o oracle cair, compras no contrato direto (sem passar pelo backend) não serão bloqueadas para carteiras não verificadas
